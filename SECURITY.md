# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| v0.3.5 (stable) | ✅ |
| v0.4.4 (beta) | 🧪 |

## Reporting a Vulnerability

If you discover a security vulnerability in GateSwarm MoA Router, please report it responsibly.

**Do not** open a public GitHub issue for security vulnerabilities.

Instead:

1. Email the maintainer privately, or
2. Open a private [GitHub Security Advisory](https://github.com/pealmeida/gateswarm-moa-router/security/advisories/new)

Please include:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

Allow **7 days** for a response and fix before any public disclosure.

## Response Timeline

- **Acknowledgment:** within 48 hours
- **Initial assessment:** within 5 business days
- **Patch release:** depends on severity (critical issues prioritized)

## Security Scope

Areas of concern:
- API key exposure in logs or error messages
- Prompt data leakage through feedback loops
- Container image vulnerabilities
- Dependency supply chain attacks

This policy applies to:
- `router.py` — production scoring engine
- `train.py` — training pipeline
- `llmfit/` — dataset factory toolkit
- `v32_cascade_weights.json` — pre-trained weights

Not a concern (out of scope):
- Third-party model provider APIs (report to respective providers)
- Content generated by upstream LLM providers
- Prompt content moderation (GateSwarm is a router, not a guardrail)
- Issues in dependencies (numpy, scipy, scikit-learn) — report upstream

## Acknowledgments

We appreciate responsible disclosure and will credit researchers who report valid security issues.
